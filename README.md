# Digital Factory — Цифровой завод

Windows-приложение для оркестрации данных между промышленными системами (Vault PDM, Tecnomatix Plant Simulation, AutoCAD, Excel): автоматизирует пайплайн **PDM → импорт → симуляция → отчёт** и помогает команде проверять гипотезы по компоновке завода без ручного перекладывания данных.

## Установка

Скачайте последний релиз из `release/`:

```bash
# Распакуйте архив и запустите установщик
Digital_Factory_v0.3.0.zip → setup.exe
```

Установка выполняется в профиль текущего пользователя — права администратора не нужны.  
Если WebView2 отсутствует на машине, установщик скачает его автоматически (~2 МБ).

**Требования:**
- Windows 10 / 11 (64-bit)
- Tecnomatix Plant Simulation 16 (установленный в `C:\Program Files\Siemens\`)

## Быстрый старт

1. Запустите `setup.exe` и установите приложение.
2. Откройте **Digital Factory** из меню Пуск (окно запускается развёрнутым).
3. Нажмите шестерёнку (⚙) в правом верхнем углу и укажите:
   - Путь к ярлыку Plant Simulation (`.lnk`)
   - Путь к файлу модели (`.spp`)
   - Имя SimTalk-метода
   - URL сервера Vault PDM + токен (или оставьте пустым для mock-режима)
4. Для каждой карточки этапа выберите режим **Тест** или **Реал** переключателем.
5. Нажмите **Запуск Цифрового завода** в нижней части экрана — пайплайн выполнится последовательно.

## Использование

Интерфейс разделён на две панели с регулируемой шириной (20–80%):

- **Левая панель** — аккордеон из трёх шагов: Импорт, Симуляция, Отчёт.
- **Правая панель** — вкладки **КОНСОЛЬ** (лог событий) и **ОТЧЁТ** (карточки с результатами).

### Шаг 1 — Импорт исходных данных

| Источник | Что загружается |
|---|---|
| **Vault PDM** | Состав изделия (BOM) по номеру детали; сохраняется в `bom.json` и конвертируется в `bom.xml` |
| **Excel** | Нормативы и таблицы (mock) |
| **AutoCAD** | Планировочные решения (mock) |

Каждая карточка имеет переключатель **Тест / Реал**, который определяет режим выполнения. В тест-режиме данные заменяются mock-значениями без обращения к внешним системам.

### Шаг 2 — Симуляция производства

После завершения всех этапов импорта активируется шаг симуляции:

- **Tecnomatix Plant Simulation** — запускается через `.lnk`-ярлык с параметрами из настроек (путь к `.spp`-модели и имя SimTalk-метода). По завершении расчёта приложение читает `results.txt` из записываемой директории.
- **Visual Components** — карточка зарезервирована, недоступна (Coming soon).

### Шаг 3 — Отчёт

После симуляции на вкладке **ОТЧЁТ** правой панели автоматически отображаются карточки с результатами. Формат динамический: каждая пара `ключ=значение` из `results.txt` становится отдельной карточкой — количество метрик неограниченно.

Консоль одновременно выводит те же данные в текстовом виде.

### Кнопка запуска

Кнопка **Запуск Цифрового завода** в нижней части окна запускает все этапы последовательно в порядке: PDM → Excel → AutoCAD → Plant Simulation. Во время выполнения кнопка переключается в **Остановить**.

## Настройки

Все настройки сохраняются в `settings.json` рядом с исполняемым файлом (или в `%APPDATA%\Digital Factory\`):

| Параметр | Описание |
|---|---|
| `plant_sim_shortcut` | Путь к `.lnk`-ярлыку Plant Simulation |
| `spp_path` | Путь к файлу модели Plant Simulation (`.spp`) |
| `sim_method` | Имя SimTalk-метода для автозапуска (например, `.UserObjects.printed`) |
| `vault_url` | URL Vault PDM API (`http://host:port`); пусто = mock-режим |
| `vault_token` | Токен для авторизации в Vault (заголовок `token`) |
| `vault_part_number` | Обозначение изделия по умолчанию |

Промежуточные файлы (`bom.json`, `bom.xml`, `results.txt`) сохраняются в первую записываемую директорию из списка: рядом с `.exe` → `%APPDATA%\Digital Factory\` → `%LOCALAPPDATA%\Digital Factory\`.

Соотношение ширины панелей сохраняется в `localStorage` (ключ `panelLeftPct`).

## Архитектура

```
bratsy-tauri/
├── src-tauri/src/lib.rs   # Rust-бэкенд: Tauri-команды
│   ├── run_plantsim        # Запуск Plant Simulation через .lnk, чтение results.txt
│   ├── vault_get_bom       # HTTP-запрос BOM из Vault PDM (или mock), сохранение bom.json
│   ├── bom_to_xml          # Конвертация bom.json → bom.xml
│   ├── run_stage           # Mock-запуск для Excel / AutoCAD
│   ├── stop_stage          # Остановка запущенного этапа (taskkill по PID)
│   ├── pick_file / pick_folder  # Нативный диалог выбора файла (PowerShell)
│   └── find_plantsim_shortcut   # Резолвинг пути к .lnk из настроек
├── src/
│   ├── index.html          # Двухпанельный UI: левый аккордеон + правая панель (Console/Report)
│   ├── main.js             # Pipeline state machine, resize handle, Tauri invoke-вызовы, localStorage
│   └── styles.css          # Тёмные/светлые карточки, переключатели, вкладки правой панели
└── src-tauri/tauri.conf.json   # Конфиг: productName, version 0.3.0, NSIS bundling, maximized: true
```

**Стек:** Rust + Tauri v2 (бэкенд), Vanilla HTML/CSS/JS + WebView2 (фронтенд), NSIS (дистрибуция).
