# Начало работы — Digital Factory (DP_orchestra) v0.3.0

## Требования

Перед установкой убедитесь, что выполнены следующие условия:

- **Windows 10 / 11 (64-bit)** — приложение работает только на Windows
- **Tecnomatix Plant Simulation** — требуется для шага симуляции; стандартный путь установки: `C:\Program Files\Siemens\Tecnomatix Plant Simulation 16\PlantSimulation16.exe`
- **WebView2** — загружается автоматически (~2 МБ) во время установки, если отсутствует
- Права администратора **не требуются** — установка выполняется для текущего пользователя (`%LOCALAPPDATA%\Programs\Digital Factory\`)

## Установка

### Шаг 1 — Скачайте дистрибутив

Перейдите в раздел [Releases](https://github.com/ney9992/DP_orchestra/releases) и скачайте архив актуальной версии:

```
Digital_Factory_v0.3.0.zip
```

### Шаг 2 — Распакуйте архив

Разархивируйте скачанный `.zip` в любую папку.

### Шаг 3 — Запустите установщик

```
setup.exe
```

Если WebView2 не установлен — он загрузится и установится автоматически (~2 МБ). Дополнительного вмешательства не требуется.

### Шаг 4 — Запустите приложение

После установки откройте **Digital Factory** из меню Пуск. Окно открывается максимизированным автоматически.

## Первый запуск

При первом запуске необходимо настроить параметры в панели настроек.

### 1. Создайте ярлык Plant Simulation вручную

Перед открытием настроек создайте ярлык Windows (`.lnk`), указывающий на исполняемый файл Plant Simulation:

1. Найдите `PlantSimulation16.exe` (обычно в `C:\Program Files\Siemens\Tecnomatix Plant Simulation 16\`)
2. Кликните правой кнопкой мыши → **Отправить → Рабочий стол (создать ярлык)**
3. Переместите ярлык в удобное место (например, в папку документов)

> Приложение записывает аргументы запуска в поле **Arguments** этого ярлыка непосредственно перед каждым прогоном симуляции.

### 2. Откройте настройки

Нажмите значок шестерёнки (&#9881;) в правом верхнем углу панели управления.

### 3. Заполните раздел PLANT SIMULATION

| Поле | Обязательно | Описание |
|------|-------------|----------|
| **Путь к ярлыку (.lnk)** | Да | Путь к `.lnk`-файлу Plant Simulation. Нажмите `…` для выбора файла через диалоговое окно. |
| **Путь к модели (.spp)** | Да | Путь к файлу модели Plant Simulation (`.spp`). Нажмите `…` для выбора. Если не задан, запуск симуляции завершится ошибкой. |
| **Метод SimTalk** | Да | Имя SimTalk-метода, который запускает симуляцию (например, `.UserObjects.printed`). Последнее использованное значение запоминается между сессиями. |

### 4. Настройте Vault PDM (опционально)

| Поле | Описание |
|------|----------|
| **URL сервера** | Адрес Vault PDM API (например, `http://192.168.1.100:8080`). Оставьте пустым — приложение будет работать в mock-режиме с тестовыми данными BOM. |
| **Токен авторизации** | Токен для доступа к Vault API. Передаётся в заголовке `Authorization: token {token}`. |
| **Обозначение изделия** | Обозначение детали для запроса BOM (например, `МЧД-001`). Если пустое, используется `МЧД-001` из mock-данных. |

### 5. Сохраните настройки

Нажмите **Сохранить**. Настройки записываются в файл `settings.json` рядом с исполняемым файлом.

## Работа с пайплайном

Пайплайн запускается единой кнопкой **«Запуск Цифрового завода»** в нижней части окна.

### Шаг 1 — Выберите режим для каждого источника данных

На каждой карточке источника данных есть переключатель **Тест / Реал**:

- **Тест** — имитирует работу этапа с mock-данными (по умолчанию). Не требует подключения к внешним системам.
- **Реал** — выполняет реальную интеграцию (запрос к Vault API, запуск скрипта, запуск Plant Simulation).

Источники данных:

| Карточка | Описание |
|----------|----------|
| **Vault PDM** | Запрос BOM (состав изделия). В режиме Реал обращается к Vault API; BOM сохраняется как `bom.json`, затем конвертируется в `bom.xml`. |
| **Excel** | Загрузка нормативов и данных ТРП. |
| **AutoCAD** | Загрузка планировочных решений цеха. |
| **Tecnomatix Plant Simulation** | Запуск симуляции производственной линии. |

### Шаг 2 — Нажмите «Запуск Цифрового завода»

Кнопка запускает все этапы последовательно: Шаг 1 (импорт трёх источников) → Шаг 2 (симуляция). Ход выполнения отображается в левой панели в виде аккордеона шагов и на статусных пилюлях каждой карточки.

Во время выполнения кнопка меняет текст на **«Остановить»** — нажмите её, чтобы прервать пайплайн.

### Шаг 3 — Просмотрите отчёт

После завершения симуляции вкладка **ОТЧЁТ** в правой панели открывается автоматически. Отчёт отображает все пары `ключ=значение` из файла `results.txt` в виде карточек — набор метрик определяется вашей SimTalk-моделью.

Для нового прогона нажмите **«Запуск Цифрового завода»** повторно — пайплайн и отчёт сбрасываются автоматически.

### Ширина панелей

Разделительную полосу между левой и правой панелями можно перетащить мышью. Соотношение ширин сохраняется между сессиями (диапазон: 20 %–80 % от ширины окна).

## Типичные проблемы при первом запуске

### «Путь к ярлыку Plant Simulation не задан»

Откройте настройки (шестерёнка) и укажите путь к `.lnk`-файлу в поле **«Путь к ярлыку (.lnk)»**.

### «Путь к .spp модели не задан»

В настройках заполните поле **«Путь к модели (.spp)»**. Без него кнопка запуска завершится ошибкой на этапе Plant Simulation даже при наличии ярлыка.

### Plant Simulation не запускается

Убедитесь, что ярлык (`.lnk`) указывает на существующий `PlantSimulation16.exe`. Проверьте путь в свойствах ярлыка (Правая кнопка → **Свойства → Объект**).

### Vault PDM возвращает mock-данные вместо реальных

Ожидаемое поведение, если поле **URL сервера** пустое или содержит значение `mock`. Укажите адрес реального Vault-сервера и токен авторизации в настройках. Токен передаётся в заголовке `Authorization: token {token}` (без слова `Bearer`).

### WebView2 не устанавливается

Если сеть недоступна, скачайте установщик WebView2 вручную с сайта Microsoft и установите его до запуска `setup.exe`. <!-- VERIFY: актуальная ссылка на офлайн-установщик WebView2 -->

### `results.txt` не найден после симуляции

В консольной панели появляется предупреждение `[warning] results.txt не найден — результаты недоступны`. Убедитесь, что SimTalk-метод в вашей `.spp`-модели записывает результаты в файл `results.txt` в формате `ключ=значение` (по одной паре на строку) в записываемую директорию приложения. Набор ключей произвольный — отчёт отображает все найденные пары.
