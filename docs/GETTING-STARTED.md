# Начало работы — Digital Factory (Bratsy_DP)

## Требования

Перед установкой убедитесь, что выполнены следующие условия:

- **Windows 10 / 11 (64-bit)** — приложение работает только на Windows
- **Tecnomatix Plant Simulation** — требуется для шага симуляции; стандартный путь установки: `C:\Program Files\Siemens\Tecnomatix Plant Simulation 16\PlantSimulation16.exe`
- **WebView2** — загружается автоматически (~2 МБ) во время установки, если отсутствует
- Права администратора **не требуются** — установка выполняется для текущего пользователя (`%LOCALAPPDATA%\Programs\Digital Factory\`)

## Установка

### Шаг 1 — Скачайте дистрибутив

Перейдите в раздел [Releases](https://github.com/ney9992/Bratsy_DP/releases) и скачайте архив актуальной версии:

```
Digital_Factory_vX.Y.Z.zip
```

### Шаг 2 — Распакуйте архив

Разархивируйте скачанный `.zip` в любую папку.

### Шаг 3 — Запустите установщик

```
setup.exe
```

Если WebView2 не установлен — он загрузится и установится автоматически (~2 МБ). Дополнительного вмешательства не требуется.

### Шаг 4 — Запустите приложение

После установки откройте **Digital Factory** из меню Пуск.

## Первый запуск

При первом запуске необходимо настроить путь к ярлыку Plant Simulation.

### 1. Создайте ярлык Plant Simulation вручную

Перед открытием настроек создайте ярлык Windows (`.lnk`), указывающий на исполняемый файл Plant Simulation:

1. Найдите `PlantSimulation16.exe` (обычно в `C:\Program Files\Siemens\Tecnomatix Plant Simulation 16\`)
2. Кликните правой кнопкой мыши → **Отправить → Рабочий стол (создать ярлык)**
3. Переместите ярлык в удобное место (например, в папку документов)

> Приложение читает и модифицирует только поле **Arguments** этого ярлыка во время симуляции.

### 2. Откройте настройки

Нажмите значок шестерёнки (&#9881;) в правом верхнем углу панели управления.

### 3. Укажите путь к ярлыку

В поле **«Путь к ярлыку Plant Simulation (.lnk)»** укажите путь к созданному файлу `.lnk`. Используйте кнопку `…` для выбора файла через диалоговое окно.

### 4. Настройте Vault PDM (опционально)

| Поле | Описание |
|------|----------|
| URL сервера Vault | Адрес Vault PDM API (например, `http://192.168.1.100:8080`). Оставьте пустым для работы в демо-режиме с тестовыми данными. |
| Bearer-токен | Токен авторизации для доступа к Vault API |
| Обозначение изделия по умолчанию | Обозначение детали для запроса BOM (например, `МЧД-001`) |

### 5. Сохраните настройки

Нажмите **Сохранить**. Настройки записываются в `settings.json` рядом с исполняемым файлом.

## Работа с пайплайном

Пайплайн состоит из трёх последовательных шагов.

### Шаг 1 — Импорт данных

Кликайте по карточкам источников данных по очереди:

- **Vault PDM** — запрос BOM (состав изделия) из Vault API или демо-данных
- **Excel** — загрузка нормативов и таблиц (имитационный режим)
- **AutoCAD** — загрузка планировочных решений (имитационный режим)

После завершения всех трёх источников Шаг 2 разблокируется автоматически.

### Шаг 2 — Симуляция производства

1. Нажмите карточку **Tecnomatix Plant Simulation**
2. В диалоговом окне выберите файл модели (`.spp`)
3. Введите имя SimTalk-метода (последнее использованное значение запоминается между сессиями)

Приложение изменяет аргументы ярлыка и запускает Plant Simulation. Дождитесь завершения симуляции и закрытия окна Plant Simulation.

### Шаг 3 — Результаты расчёта

После завершения симуляции отчёт появляется автоматически и содержит 7 метрик:

| Метрика | Единица |
|---------|---------|
| Загрузка линии | % |
| Пропускная способность | ед./ч |
| Время цикла | сек |
| OEE | % |
| WIP | ед. в работе |
| Lead time | мин |
| Узкое место (Bottleneck) | название станции |

Нажмите **↺ Новый расчёт** для сброса пайплайна и начала нового прогона.

## Типичные проблемы при первом запуске

### «Путь к ярлыку Plant Simulation не задан»

Откройте настройки (шестерёнка) и укажите путь к `.lnk`-файлу в поле **«Путь к ярлыку Plant Simulation (.lnk)»**.

### Plant Simulation не запускается

Убедитесь, что ярлык (`.lnk`) указывает на существующий `PlantSimulation16.exe`. Проверьте путь в свойствах ярлыка (Правая кнопка → **Свойства → Объект**).

### Vault PDM возвращает демо-данные вместо реальных

Это ожидаемое поведение, если поле **URL сервера Vault** пустое или содержит значение `mock`. Укажите адрес реального Vault-сервера и Bearer-токен в настройках.

### WebView2 не устанавливается

Если сеть недоступна, скачайте установщик WebView2 вручную с сайта Microsoft и установите его до запуска `setup.exe`. <!-- VERIFY: актуальная ссылка на офлайн-установщик WebView2 -->

### `results.txt` не найден после симуляции

В лог-панели появляется предупреждение `[warning] results.txt не найден — результаты недоступны`. Убедитесь, что SimTalk-метод в вашей `.spp`-модели записывает результаты в файл `results.txt` в формате `ключ=значение` в той же директории, где находится ярлык.
